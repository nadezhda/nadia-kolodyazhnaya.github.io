Booking.com Accessibility and Usability Evaluation

Accessibility

Several pages of the website have been evaluated for accessibility. The website has many issues, which can make it inaccessible for some people with disabilities or those who has poor connection. However, some assistive tools as ATbar tend to improve accessibility of resource.

# Test Results
1 Page title Website has inaccessible page title which consists extra information and makes it more difficult to understand for people with visual impairments. According to WAI guidelines, it is suggested that new information appear at the beginning of the page title.
2 Images ALT Attributes Alternative text for images is appropriate and understandable, but not all images contain it. For instance, one of the most usable icons for changing the language does not contain alternative text. Existing alt attribute for the title has the same color as a background and cannot be seen with turned off pictures on the page.
3 Appropriate use of tables Tables used for design of website
4 Headings All web pages do not contain <h1> and <h2> headers, but <h3>, <h4> and <h5> headings are used
5 Luminance contrast ratio The web site contains combinations of fonts and backgrounds which difficult to distinguish
6 Resize the text Text on the website can be easily resized, and the text and images do not overlap. All blocks of website depending on font size are displayed correctly
7 Keyboard access and visual focus The website provides key navigation and each element can be tabbed in a certain sequence. However, the pop-up elements such as calendars are unavailable for keyboard. Moreover, cursor is not highlighted sufficiently and sometimes unobvious where it is placed.
8 Form, Label and Errors All form elements are accessible from the keyboard. The fields are not marked as required, but all of them should be filled. The fields are provided with an appropriate error explanation. The automatic checker showed that form is not completely accessible and contains errors. Some fields do not have label attributes. Instead label attributes headings are used (<h3> for Check-in date and Check-out date, and <h5> for Guests). Not all <input> elements contain “id” attributes.
9 Login and signup forms Forms are not accessible from screen readers. Tab “Sign up” is not accessible from the keyboard
10 Removal of stylesheet and images Information appears in order and available for screen readers

Usability

In spite of breaches in accessibility, website seems quite usable, graphical user interface is friendly, and it provides services are convenient to use.

# Test Results
1 Site purposes Name of the website has an appropriate size and noticeable location. Site name clearly defines the purposes. Searching form for booking the hotels is located on the Home page and brightly highlighted. Users do not need waste time reading unnecessary information
2 Communicating Information About Your Company Website provides search and reservation hotels services and does not require extra information such as “About us” or “Contacts”
3 Content Writing Important sections are well emphasized. The combination of clear text and icons allow users to avoid confusion on the website. The webpages do not contain redundant information. The well-highlighted headers make it easier to navigate on the pages
4 Revealing Content Through Examples Each example on the webpages consists direct link which goes directly to the detailed information
5 Archives and Accessing Past Content The website provides a section of recent viewed cities and hotels
6 Links Links on the website have understandable text and do not confuse users. Several types of links are used on the website and some of them are highlighted with color to show visited and unvisited links. However, this mechanism is not applied for all types of links.
7 Navigation The particular navigation does not exist on the website
8 Search A search form contains several fields to fill and is available on all pages of the website. Despite the complexity, the form is understandable and easy to use. Text fields are long enough to type there requested information
9 Tools and Task Shortcuts The website has some quick buttons which allow to users quickly adjust requested information
10 Graphics and Animation Presented images illustrate real content and icons used on the website are clear. Animation is not used on the website
11 Graphic Design Font styles are limited, but some text has low contrast and merges with background. The website is correctly displayed on the screen with a different resolution avoiding horizontal scroll
12 URLs URL of the home page is simple and memorable
13 Popup Windows and Staging Pages Pop-up windows are used on the website, but it does not distract usability of the website. Also, routing pages mechanism is used on the website and site appearance directly in a particular language depending on user geographical location
14 Advertising The website does not provide advertising
15 Page Reload and Refresh The website does not reload and refresh the content automatically
16 Customization The website saves customizable parameters of users for subsequent visits
17 Dates and Times Dates and times definitions are provided in legible form. Users can see that information updated on the website

Universal Design Tests

# Test Results
1 Simplicity The content of the website looks simple and understandable. All essential information and options attracts the eye and makes information accessible
2 Central Layout The website is positioned centrally
3 Fewer Columns 2 columns design is used
4 Separate top section Title is clearly separated from the rest of the webpage
5 Solid areas of screen real-estate All areas of website can be distinct clearly from each other
6 Simple navigation The website does not contain particular navigation, but quick buttons are used
7 Bold logos Bold logo is used on the website. It is clear and makes the first impression about the website and content
8 Bigger text Some text is emphasized on the webpage, but some information has quite small font size and merge with background
9 Bold text introductions Bold captions are used on the website to introduce the sections
10 Strong colors Some bright colours emphasize important data and forms, however some text merge with and background
11 Rich surfaces 3D effects are used on the website
12 Gradients No gradient effects are used on the website
13 Reflections Reflection effects are used for the buttons on the website
14 Cure icons Icons on the website have good-looking design, recognizable and have a clear meaning
15 Star flashes No star flashes used on the website
